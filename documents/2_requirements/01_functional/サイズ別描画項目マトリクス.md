## 注記（シート原文より）

- テキスト項目 **"N/A"** は、原稿デザイン内に項目を設けない。**AIツールにおいても、N/A項目のデータは取得せず、AIにも渡さない**
  - → サイズ別の入力制約はAPI側の400バリデーションではなく、クライアント（Bubble）側の出し分け責務。N/Aのフィールドはリクエストに含めないこと
- テキスト項目 **"任意"** は、ユーザーがテキスト入力しないケースを許容する。テキスト未入力で画像生成する場合、未入力の項目は `""`（空文字）にして画像生成する
- 補足: 原稿画像内に存在しない項目のテキストをAIに渡した場合、どのようなレイアウトの画像になるかは**未検証**。現状仕様では登録する元画像データを間違えない限り、この現象は起きないはず

## テキスト項目の初期値

| 項目 | 初期値 |
|---|---|
| 行事名 | 管理者が行事登録時に設定 |
| 行事詳細（任意） | 管理者が行事登録時に設定 |
| プログラム（任意） | 管理者が行事登録時に設定 |
| 注釈（任意） | 管理者が行事登録時に設定 |
| 連絡先名 | ユーザーの「企業情報＞企業名」を流用する |
| 連絡先詳細（任意） | ユーザーの「企業情報＞住所、電話番号、メールアドレス」を流用する |

- ユーザーがUI上で各項目の内容を変更して画像生成した場合はその内容が保持される。ただし、マスターである企業情報（企業名、住所、電話番号、メールアドレス）には反映しない
- 企業情報変更時の行事への反映:
  - 開催後の行事は「連絡先名」「連絡先詳細」を変更しない
  - 開催前、かつ、行事の連絡先情報と（変更前の）企業情報が一致している場合のみ「連絡先名」「連絡先詳細」を自動更新する
  - 開催前、かつ、連絡先情報が（変更前の）企業情報から変更されている場合はユーザーの意図とみなし、自動更新しない
  - 新規作成された行事の「連絡先名」「連絡先詳細」は変更する
  - 表記揺れ（スペース・ハイフン等）は今回考慮外とし、対応しない
