# 用語集

| 用語 | 説明 |
|------|------|
| POC | Proof of Concept。本番開発前の検証フェーズ |
| 行事 | 業界ごとのイベント（例: 法要、入学式等）。開始日・終了日で管理 |
| 行事テンプレート | 行事の雛形。デザインテンプレートを1〜3つ紐付ける |
| デザインテンプレート | デザイン+文言のセット。サイズごとに画像を持つ |
| 背景テンプレート | デザインの背景画像。行事テンプレートとは分離管理 |
| 業界 | サービス対象の業種カテゴリ（最大20程度）。2階層管理（例: お寺→宗派） |
| 原稿 | 印刷物デザインデータ。APIで自動生成される |
| 共通印刷物 | 全業界共通/業界別共通/第2階層共通 の3分類で管理 |
| ノベルティ | オリジナルグッズ（巾着、トートバッグ等）。POCでは紹介のみ |
| 長尺 | 細長い印刷物（縦・横）。バナー等に利用 |
| Bubble | ノーコード開発プラットフォーム。フロントエンド構築に使用 |
| トリガーAPI | Bubble と画像生成API の間に挟む中継API |
| デザインパターン | 1つの行事に対して生成される3種のデザイン案（A/B/C） |
| APIコネクター | Bubble側から外部APIを呼び出す機能。レスポンス上限5MB |
| Webhook | API側から処理完了を通知するコールバック方式。非同期処理の選択肢の一つ |
| 再生成 | ユーザーが明示的にボタンを押してデザインを再度生成する操作。「保存」とは分離 |
| 評価 | デザインに対する五段階評価。POCでは全デザイン強制評価 |
| プリセット | テンプレートにあらかじめ設定されたデフォルトテキスト・画像のセット |
| 注釈 | 行事の補足説明テキスト。選択式+自由入力、最大行数・最大文字数で制限 |
| プログラム | 行事の進行内容（式次第等）。注釈と同様にテキスト入力制限あり |
| 年度管理 | 業界ごとに異なる年度区切りの管理。切替タイミングは業界別に検討 |
| エージェンシープラン | Bubble の代理店向けプラン。開発中に利用、リリース時にグロースプランへ移行 |
| グロースプラン | Bubble の本番運用向けプラン。リリース時に購入予定 |
| 操作ログ | ユーザーの操作履歴。CSV出力形式、月単位でダウンロード可能 |
| リマインダー | 行事の2ヶ月前・1ヶ月前に送信されるメール通知 |
