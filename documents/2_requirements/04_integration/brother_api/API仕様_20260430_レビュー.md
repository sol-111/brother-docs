## 2. 20260417 → 20260430 の差分

### DB設計の差分

- Source Image DB: **eventName, eventPeriod, eventDetails, eventProgram, notes** の5項目が追加
- Generated Image DB: 同じく **event系5項目** が追加

### APIエンドポイントの差分

| 変更点 | 20260417 | 20260430 |
|---|---|---|
| エンドポイント数 | 7本（元原稿2 + 生成5） | **8本**（元原稿**3** + 生成5） |
| 新規追加 | — | **GET /source-images-size**（サイズ指定で原稿一覧取得） |
| POST /generated-images の入力 | sourceImageId, originalText, replaceText | sourceImageId, **eventName, eventPeriod, eventDetails, eventProgram, notes** |
| POST /generated-images のstatus | RUNNING | **PENDING** |
| DELETE の入力 | generatedImageId のみ | generatedImageId + **eventName, eventPeriod, eventDetails, eventProgram, notes** |
| GET /generated-images の入力 | userId | userId or **不要（認証から取得）** |

### シーケンス図の差分

| 変更点 | 20260417 | 20260430 |
|---|---|---|
| 登場コンポーネント | Backend APIが直接AIを呼ぶ | **Generation Worker** が間に入る |
| POSTのリクエスト内容 | originalText, replaceText | eventName, eventPeriod, eventDetails, eventProgram, notes |
| ステータス遷移 | いきなり RUNNING → COMPLETED | **PENDING → RUNNING → COMPLETED**（3段階） |
| 非同期の仕組み | APIが直接AI呼び出し後に完了通知 | APIはPENDINGを即返却 → **Workerが非同期で処理** |

## 4. その他の気になる点

### 4-1. fileName vs title の不一致

- DB設計では `fileName` というカラムが定義されている
- APIレスポンスでは `title` というフィールドを返す
- 同じものの命名揺れなのか、別フィールドなのか不明
- DB側に `title` がないまま API が `title` を返すのは仕様として不整合

### 4-2. Generated Image DB に thumbnailKey がない

- `GET /generated-images` のレスポンスには thumbnailUrl が含まれている
- しかし DB2（Generated Image DB）に thumbnailKey カラムが定義されていない
- DB1（Source Image DB）には thumbnailKey があるのに、DB2にはない
- DB設計の記載漏れではないか？

### 4-3. replaceText の位置付けが曖昧

- DB2に `replaceText` カラムがある
- しかし初回生成（POST /generated-images）では replaceText を送らず、event系フィールドを送る
- regenerateでは originalText + replaceText を送る
- 初回生成時に replaceText に何が入るのか？空？event系フィールドを結合した文字列？

### 4-4. Source Image DBのevent系5項目はBubble側では使用しない

- Source Image DB に eventName, eventPeriod, eventDetails, eventProgram, notes があるが、Bubble側では一切使用しない
- 実害はないが、認識を合わせておく

### 4-5. GET /source-images-size のパス設計が不自然

- `/source-images-size` というパスは REST 的に不自然
- 通常は `GET /source-images?size=A4` のようにクエリパラメータで絞り込むべき
- 既存の `GET /source-images` にsizeパラメータを追加すれば済む話では？
- なぜ別エンドポイントとして切り出しているのか意図不明
