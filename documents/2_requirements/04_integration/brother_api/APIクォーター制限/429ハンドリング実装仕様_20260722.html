2. リトライ仕様まとめ

トリガー（判別文字列）リトライ間隔上限上限到達時
① 429 + Too Many Requestsする（同一リクエスト再送）5秒 仮1回 仮制限中モードへ
② 429 + Limit Exceededしない——即・制限中モードへ
③ FAILED + AI_REQUEST_FAILED:429する（再生成POST）30秒 仮2回 仮「混み合っています」表示
その他（403 / 5xx / タイムアウト）しない——汎用エラー表示（§4）
実装ポイント: ①②はAPI Connector呼び出し自体の失敗（ワークフローのエラーハンドリング＝ An unhandled error occurs イベント等で Current Workflow Error's message の文字列を判定）。 ③は正常応答（200）の中身で判定する。レイヤーが違う点に注意。

3. ポーリングのガード（クォータ消費対策・必須）

今回のクォータ枯渇の最有力原因はポーリングの回りっぱなし。以下を必須要件とする。

  • 間隔10秒、最大18回（3分）でタイムアウト停止（正常時の生成は10〜60秒／件）
  • 全件 COMPLETED / FAILED で停止
  • ページ内イベント（Do every X seconds）で実装し、Backend workflowの再帰スケジュール禁止（離脱後も回り続けクォータを消費するため）
  • ポーリング中に②を検知したら即停止 → 制限中モードへ

4. ユーザー向け文言一覧

場面文言
② 制限中モードのバナー（暫定）
現在アクセスが集中しているため、デザインの生成・ダウンロードを一時停止しています。お手数ですが、時間をおいてお試しください。
リセットタイミング判明後（例: 日次・毎日0時）:
本日のデザイン生成回数の上限に達しました。明日以降に再度お試しください。
③ 自動リトライ上限超過
生成が混み合っています。しばらく待ってから、もう一度お試しください。
その他エラー（403 / 5xx / タイムアウト）
エラーが発生しました。時間をおいて再度お試しいただくか、解消しない場合は管理者にお問い合わせください。

5. テスト観点

②はブラザー側にクォータを一時的に極小値へ変更してもらうのが確実。①③は通常再現が難しいため、API ConnectorのURLを固定429を返すモックに一時差し替えて確認する方法を推奨。

  • ②発生時: 生成・再生成・DLボタンが無効化され、バナーが表示される
  • ②発生時: 行事一覧・作成済みデザイン閲覧・サムネイルは正常動作する
  • ②発生時: リトライによる追加リクエストが発生しない（Bubbleログで確認）
  • ②解除後: ページロードで制限中モードが解除されボタンが復活する
  • ①発生時: ユーザーに見えない形で1回再送され、成功すれば通常フローに戻る
  • ③発生時: 再生成が最大2回自動実行され、3回連続失敗で文言表示
  • ポーリング: 全件完了で停止／3分でタイムアウト停止／ページ離脱後にリクエストが継続しない（ログ確認）
  • DL: URL取得後すぐに署名付きURLが開かれる（600秒期限内）
