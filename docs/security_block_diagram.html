2. データフロー

2.1 デザイン生成フロー

sequenceDiagram
    actor User as ユーザー
    participant FE as Bubble Frontend
    participant API as Brother Backend API
    participant SIDB as Source Image DB
    participant GIDB as Generated Image DB
    participant Worker as Generation Worker
    participant AI as Generative AI
    participant FS as File Storage

    User->>FE: 「生成」ボタン押下
    FE->>API: POST /generated-images
{ sourceImageId, eventName, ... }
    Note over FE,API: HTTPS（TLS 1.2+）

    API->>SIDB: sourceImageId 存在確認
    SIDB-->>API: OK

    API->>GIDB: 生成レコード作成
status = PENDING
    GIDB-->>API: generatedImageId

    API->>Worker: 非同期生成処理を開始
    API-->>FE: generatedImageId, status=PENDING
    FE-->>User: 「生成中」を表示

    Worker->>GIDB: status = PROCESSING
    Worker->>AI: 画像生成依頼
    AI-->>Worker: 生成画像データ

    Worker->>FS: 生成画像を保存
    FS-->>Worker: fileKey

    Worker->>GIDB: status = COMPLETED
fileKey 更新

    User->>FE: 生成結果を確認（ポーリング）
    FE->>API: GET /generated-images/{id}
    API->>GIDB: ステータス取得
    GIDB-->>API: status=COMPLETED, fileKey
    API->>FS: 表示用URL生成
    FS-->>API: 署名付き一時URL
    API-->>FE: status=COMPLETED, imageUrl
    FE-->>User: 生成画像を表示
図2: デザイン生成フロー — 非同期生成 + ポーリングによるステータス確認

sequenceDiagram

2.3 認証フロー

sequenceDiagram
    actor User as ユーザー
    participant FE as Bubble Frontend
    participant BDB as Bubble DB
    participant MS as メール送信

    Note over User,MS: 【初回登録】
    User->>FE: アカウント登録
（メール + パスワード）

    User->>FE: メール + パスワード入力
    FE->>BDB: 認証情報照合
    BDB-->>FE: 認証OK
    FE-->>User: セッション発行・ログイン完了

3. セキュリティ対策一覧

3.1 認証・認可

項目内容ステータス
認証方式メールアドレス + パスワード確定
パスワード要件英数8文字以上（記号不要）確定
メール認証アカウント作成時に実施。認証後にサービス利用開始確定
セッション管理Bubble 標準のセッション管理機能を使用確定
アカウント単位企業単位で1アカウント確定
Bubble → Brother API 認証ファイルDL・生成・再生成APIが認証対象（参照系GET APIは認証なし）。POC環境では無認証呼び出しを一時許可中で、API Gateway Authorizer 有効化により認証必須へ切替予定確定

3.2 通信セキュリティ

通信経路プロトコル備考
ユーザー ↔ BubbleHTTPS（TLS 1.2以上）Bubble PaaS 標準
Bubble ↔ Brother APIHTTPS（TLS 1.2以上）API Connector 経由
ファイルダウンロードHTTPS署名付き一時URL（有効期限あり）

3.3 データ保護

データ種別保管場所アクセス制御備考
ユーザー情報Bubble DB認証済みユーザーのみBubble PaaS セキュリティポリシー準拠
行事情報Bubble DB自アカウントのみ—
元原稿画像Brother File StorageRead OnlyBrother側で事前登録
生成画像Brother File Storage生成ユーザーのみ一時保管。長期保存は今後検討
サムネイル画像Brother File Storage公開URL（期限なし）Bubble DB に保存して再利用
DL用画像Brother File Storage署名付き一時URL有効期限付きで発行
操作ログBubble DB管理者のみCSV出力可ダウンロード・評価ごとに記録

3.4 アクセス制御

対象一般ユーザー管理者
行事の作成・編集・削除自アカウントのみ全アカウント
デザイン生成・ダウンロード自アカウントのみ全アカウント
Source Image DBRead Only（一覧・詳細取得）Read Only
Generated Image DB自ユーザーのデータのみ全データ
操作ログCSV出力不可可
テンプレート管理不可可
イベント終了後のデータ該当デザインのダウンロード・閲覧不可

5. APIエンドポイント別セキュリティ

エンドポイントメソッド認証入力検証備考
/source-image-groupsGETなし—ソースイメージグループ一覧取得
/source-image-groups/{id}/source-imagesGETなしID形式検証グループ内の元原稿一覧（サイズ別）
/generated-imagesPOSTあり※sourceImageId存在確認
サイズ別許可フィールド検証		非同期生成。即時にgeneratedImageId返却
/generated-images/{id}GETあり※ID形式検証ポーリングによるステータス確認
/generated-images/{id}/regeneratePOSTあり※ID形式検証再生成。新規IDを発行
/files/url/{fileKey}GETあり※fileKey形式検証署名付き一時URL発行
/common-print-groupsGETなし—共通印刷物グループ一覧
/common-print-groups/{id}/printsGETなしID形式検証グループ内印刷物一覧
  • ※ 認証対象はファイルDL・生成・再生成APIの3系統。参照系GET API（元原稿・共通印刷物）は認証なし
  • ※ POC環境では現在、無認証呼び出しを一時許可中（AllowUnauthenticatedGeneratedApi=true）。API Gateway Authorizer の有効化により認証必須へ切替予定
